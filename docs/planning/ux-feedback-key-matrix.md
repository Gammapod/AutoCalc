## Key Move/Swap Matrix

| Interaction Path | Current Feedback | Status | Notes |
|---|---|---|---|
| Drag hover over valid target | Explicit target highlight (`drop-target-valid`) | Present | Mouse + touch both supported. |
| Drag hover over invalid target | Explicit invalid highlight (`drop-target-invalid`) | Present | Mouse + touch both supported. |
| Drag carry source key | Source + ghost visuals (`drag-source`, `key--carry-source`, `drag-ghost`) | Present | Visual affordance only, no semantic reason text. |
| Successful move/swap/install/uninstall commit | Layout/result changes visible after dispatch | Partial | No standardized success text/cue taxonomy yet. |
| Drop cancel/noop (no valid target) | No explicit rejection message | Missing | Touch returns `canceled`/`noop`, but no user-visible reason copy. |
| Layout commit reject at reducer (`evaluateLayoutDrop` deny -> state unchanged) | No explicit rejection message | Missing | Reasons exist in domain (`invalid_source`, `locked_key_immobile`, etc.) but are not surfaced. |
| Move/swap reject due execution mode gate | Display reject blink (`execution_gate_rejected`) | Partial | Cue is non-specific; no rejection reason text. |
