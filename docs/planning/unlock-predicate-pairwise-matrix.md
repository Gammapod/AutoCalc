# Unlock Predicate Pairwise Matrix

Purpose: provide a pairwise planning matrix across all known predicate types, including currently implemented predicates and proposed (not-yet-implemented) predicates.

Canonical sources:

* Implemented analyzers: `src/domain/unlockEngine.ts` (`analyzers` map).

* Predicate type definitions: `src/domain/types.ts` (`UnlockPredicate` union).

* Proposed visualizer predicates: `docs/planning/visualizer-roll-content-interaction-matrix.md` (Visualizer Hint Capability Matrix).

Legend:

* `--`: diagonal (predicate paired with itself).

* `ii`: both predicates are implemented.

* `ip`: mixed pair (one implemented, one proposed).

* `pp`: both predicates are proposed (not yet implemented).

* `R:`: minimal satisfying roll/orbit sequence example.

* `A:`: minimal satisfying action-trace/state example.

## Diagonal Example Assumptions

* Length defaults: use `2` unless predicate semantics require a longer minimum.

* Numeric defaults: use smallest non-trivial values (`1`, `-1`, `0`) that satisfy the predicate.

* Equality/sequence defaults: prefer short integer orbits.

* Complex/vector defaults: use simple axis-aligned or quadrant-aligned points where possible.

* Proposed geometry-sequence predicates follow visualizer-intent semantics from `visualizer-roll-content-interaction-matrix.md`.

## Notes

* This matrix captures implementation status pairing only; it does not yet encode semantic compatibility/contradiction rules between predicate pairs.

* If needed, add a follow-up semantic layer (`compatible`, `conflicting`, `redundant`, `independent`) on top of this same index.
