# Fresh-Context Handoff Checklist

## Sender Readiness

- [ ] Objective is one sentence and testable.
- [ ] Success criteria are explicit.
- [ ] Locked decisions are complete.
- [ ] File list is explicit and minimal.
- [ ] Out-of-scope items are explicit.
- [ ] Required output schema is present.
- [ ] Delegation mode is set (`review` or `implementation`).
- [ ] Spawn config will use `fork_context: false`.

## Receiver Acceptance

- [ ] Response contains required sections in exact order.
- [ ] Findings cite provided inputs.
- [ ] Risks are split into blocking/non-blocking.
- [ ] Plan is decision-complete.
- [ ] Test plan is executable.
- [ ] Open questions are minimal and concrete.
