# v0.8.4 Refactor Log

Last updated: 2026-03-17

## Track 1: AppServices + composition root cutover

- Added `AppServices` contract and registry (`src/contracts/appServices.ts`).
- Composition root now configures services in bootstrap via `setAppServices(...)`.
- Production runtime code no longer reads `getContentProvider(...)`; source boundary test added.
- Legacy `contentRegistry` remains for scoped adapter contract tests only.

## Track 2: Action/Event/Reducer parity hardening

- Kept canonical command boundary (`executeCommand -> eventFromAction -> applyEvent -> reducer`).
- Upgraded parity suites:
  - `contracts/parity-seeded-fuzz` now executes deterministic action streams with parity assertions.
  - `contracts/parity-long-traces` now executes fixture traces with parity assertions.
- Added boundary contract doc: `docs/contracts/action-event-reducer-boundary.md`.

## Track 3: UI runtime typing cleanup

- Replaced untyped module runtime bags with typed runtime slots:
  - calculator, storage, input, visualizerHost, grapher, shell.
- Updated runtime registry reset/dispose to typed lifecycle fields.
- Removed consumer-side `runtime.state.<...> as ...` casting patterns in migrated modules.

## Temporary adapters/shims

- Retained legacy `contentRegistry` only for explicit contract coverage (`contracts/content-provider-wiring`).
- No new production shims introduced.

## Cleanup/removal checkpoints

- Removed production global content provider reads from `src/`.
- Added `contracts/content-registry-boundary` to block regressions.
- Kept behavior parity suites and strengthened execution coverage without changing gameplay semantics.
